INNER JOIN

An INNER JOIN is a CROSS JOIN with some result rows removed by a conditoin in the query.
  • EQUIJOIN
  • NON-EQUIJOIN
  • NATURAL JOIN
* EQUIJOIN SELECT column_name(parent_table_name.column_name,child_table_name.column_name) FROM parent_table_name
INNER JOIN child_table_name
ON column_name of parent_table = column_name of child_table;
* NON-EQUIJOIN <> SELECT column_name(parent_table_name.column_name,child_table_name.column_name) FROM child_table_name
INNER JOIN parent_table_name
ON column_name of parent_table <> column_name of child_table;
* NATURAL JOIN <> SELECT column_name(parent_table_name.column_name,child_table_name.column_name) FROM child_table_name
NATURAL JOIN parent_table_name;

OUTER JOIN

An OUTER JOINS returns all rows from one of the tables, along with matching information with matching information from another table.
  • LEFT OUTER JOIN/LEFT JOIN
  • RIGHT OUTER JOIN/RIGHT JOIN
  • FULL OUTER JOIN/FULL JOIN
* LEFT OUTER JOIN/LEFT JOIN The LEFT JOIN keyword returns all rows from the left table, with the matching rows in the right table. The result is NULL in the right side when there is no match.
In a LEFT OUTER JOIN the table that comes before the join is the left table, and the table that comes after the join is the right table.
Syntax:-
SELECT column_name(parent_table_name.column_name,child_table_name.column_name) FROM parent_table_name
LEFT JOIN child_table_name
ON column_name of parent_table = column_name of child_table;
* RIGHT OUTER JOIN/RIGHT JOIN The RIGHT JOIN keyword returns all rows from the fight table, with the matching rows in the left table. The result is NULL in the left side when there is no match.
In a RIGHT OUTER JOIN the table that comes before the join is the Right table, and the table that comes after the join is the Left table.
Syntax:-
SELECT column_name(parent_table_name.column_name,child_table_name.column_name) FROM parent_table_name
RIGHT JOIN child_table_name
ON column_name of parent_table = column_name of child_table;
* FULL OUTER JOIN/FULL JOIN The FULL OUTER JOIN returns all rows from the left table and from the right table.
The FULL OUTER JOIN combines the result of both LEFT and RIGHT joins.
Syntax:-
SELECT column_name(parent_table_name.column_name,child_table_name.column_name) FROM child_table_name
NATURAL JOIN parent_table_name;

UNION

A UNION combines the results of two or more quires into one table.
Rules:-
  1. The number if column in each SELECT statment must match.
  2. The data types in the columns should be same.
  3. The columns in each SELECT statement must be in the same order.
  4. Must have the same expressions and aggregate function in each SELECT statement
  5. The UNION operator selects only distinct values by default and It doesn't select duplicate values.
Ex:-
SELECT name FROM student
UNION
SELECT name FROM employee;

UNION ALL

UNION ALL returns every match whether it is duplicate or distinct ones.

INTERSECT

INTERSECT retuns only those columns that are in the first query and also in the second query.
SELECT name FROM student
INTERSECT
SELECT name FROM Employee;

EXCEPT

EXCEPT returns only those columns that are in the first query but not in the second query.
SELECT name FROM student
EXCEPT
SELECT name FROM Employee;
